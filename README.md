# research

Agent-driven research. This repository collects structured summaries and
question-and-answer writeups about open-source projects, tools, and technical
topics — produced by autonomous research agents and published here for anyone
to read.

Each report cites the sources it was built from. Reports are generated by
software, then kept as a durable, browsable record.

## What's in here

Reports live in a small, fixed set of top-level directories so the repo root
stays clean as the number of projects grows:

```
projects/                    # per-project reports, one dir per project
└── <slug>/
    ├── summary.md            # structured project/tool summary
    ├── security.md           # public-sources security posture review
    └── qa/
        └── YYYY-MM-DD-topic.md   # a batch of questions with cited answers

topics/                      # research not about a single project
└── <slug>/ …

comparisons/                 # 2–4 projects compared side by side
└── <topic-or-projects>.md
```

For example:

```
projects/nextcloud/
├── summary.md
├── security.md
└── qa/
    └── 2026-07-05-storage-backends.md

comparisons/
└── self-hosted-file-sync.md   # nextcloud vs seafile vs owncloud
```

Templates for every format live in [`templates/`](templates/). Every report
carries YAML frontmatter, and [`INDEX.md`](INDEX.md) — a browsable table of all
reports — is generated from it. See [`AGENTS.md`](AGENTS.md) for the details.

## Project catalog & category lists

Alongside the reports, [`catalog/`](catalog/) holds **machine-readable project
records** — one file per project (`catalog/<slug>.md`) with structured
frontmatter (`categories`, `features`, `tags`, `status`, `watch_*`). From those,
`scripts/build-catalog.py` generates the per-category **category lists** in
[`categories/`](categories/):

```
catalog/
└── <slug>.md            # one project record (see templates/catalog-entry.md)

categories/
├── README.md            # generated index of categories
└── <category>.md        # generated list of projects in that category
```

This is the tracking layer: agents categorise projects by feature set, keep the
`watch_*` freshness fields current, and surface what's changed. It's generated
and CI-gated the same way as `INDEX.md`.

## How reports get here

1. A subject or a list of questions is queued for a research agent.
2. The agent gathers evidence from public sources (project source code,
   documentation, and the web), then fills the relevant template.
3. The result is opened as a **pull request** — one PR per report — so the
   output is reviewable before it lands on `main`.

Questions in → cited answers out, as a PR.

## Scope — public sources only

Everything here is built **exclusively from public information** about
publicly available projects and topics. This repository deliberately contains:

- **No** private infrastructure, deployment, or operational detail.
- **No** hostnames, IP addresses, internal DNS names, or account identifiers.
- **No** secrets, credentials, or configuration.

If a report needs private context to make sense, it does not belong here.
