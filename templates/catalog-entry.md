# owner/name

Longer notes about the project — what it does, why it's on the radar, how it
compares to alternatives. Link to a full report if one exists
(e.g. `../owner-name/summary.md`).

Frontmatter field notes:

- **repo** — `owner/name` (the unique key; one catalog entry per repo).
- **url** — homepage or repo URL.
- **summary** — a single line; this is what appears in the generated awesome lists.
- **categories** — one or more; a project is listed under *each* of its categories.
- **features** — capability tags — the point of the catalog ("categorise by feature set").
- **tags** — free tags (language, runtime, etc.).
- **status** — one of: `watching` · `evaluating` · `adopted` · `rejected` · `archived`.
- **interest** — which of my interests this serves (drives what gets surfaced).
- **watch_*** — freshness state maintained by the watch loop (last release/commit/stars).
